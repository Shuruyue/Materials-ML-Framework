# ML × Materials Science Research Atlas

A first-principles research framework that maps the intersection of **machine learning** and **materials science** — from objective design through deployment — with a curated evidence base of **500+ high-quality sources**.

## What this is

This is an interactive, self-contained web page that provides:

| Section | Purpose |
|---------|---------|
| **Architecture** | Eight irreducible layers from optimization objective to deployment context |
| **Taxonomy** | Eleven deep branches covering problem formulation, evidence, representations, forward modeling, inverse design, scientific ML, characterization, manufacturing, sequential decision systems, trust/governance, and material-system verticals |
| **Frontiers** | Three growth bands: high-momentum core, adjacent corridors, and strategically underrepresented areas |
| **Bottlenecks** | Six causal chains diagnosing why ML in materials science repeatedly fails at translation |
| **Research Agenda** | Six coupled grand challenges that form a doctoral-grade roadmap |
| **Evidence Base** | 506 sources across 22 evidence clusters, filterable and linked to DOI / OpenAlex |

The framework is **bilingual** (English and Traditional Chinese) and designed to be
opened directly in any modern browser — no server, no build step, no dependencies.

## Quick start

```bash
git clone <this-repo>
cd ML
# Open in your default browser:
open index.html        # macOS
start index.html       # Windows
xdg-open index.html    # Linux
```

That's it. Everything runs client-side.

## Refreshing the evidence base

The bibliography is generated by an OpenAlex harvester. To update:

```bash
pip install -r tools/requirements.txt
python tools/build_data.py
```

This queries the [OpenAlex API](https://openalex.org/) for title-matched scholarly
works, filters by quality venue and recency, and writes the output to `data/`.

## Design principles

1. **First-principles organisation** — the framework decomposes by *what the field
   optimises*, not by which model family is trending.
2. **Stable backbone** — architecture layers and taxonomy branches change slowly;
   model names and benchmark numbers change fast.  The structure is designed to
   outlast any single model generation.
3. **Evidence-backed** — every architectural claim is pressure-tested against a
   structured bibliography spanning 2010–2026.
4. **Self-contained** — zero runtime dependencies, no CDN calls, no framework.
   Open the HTML file and it works.
5. **Bilingual** — full English and Traditional Chinese support with one-click switching.

## Technology

- Vanilla HTML / CSS / JavaScript (no frameworks, no build tools)
- CSS custom properties for theming; responsive grid layout; print stylesheet
- Python 3.10+ with `requests` for the data-harvesting pipeline
- [OpenAlex](https://openalex.org/) as the bibliometric data source
